Sentiment Analysis With Transformers

Analyzing the sentiment of Steam reviews with Transformer Enocders

 Posted on May 27, 2023  |  5 minutes  |  866 words  |  Mujtaba Bader

Project Overview

For this project I took a dataset from Kaggle consisting of game titles, user reviews, and user suggestions (1 signifying the review is positive, 0 signifying the review is negative).

EDA

I started by cleaning the data, removing any null values or duplicates, then removed any non-alphanumeric characters from the user reviews as apart of the preprocessing step before training the model. I also generated some plots to visualize some features of the data like what games had the most number of reviews,

Before saving the cleaned data I made sure to check whether the classes were imbalanced, in case I may have needed to upsample the minority class or downsample the majority class so that there would be no issue with the trained model.

Building the model

The model used is the Encoder portion of a Transformer, below is a diagram showing the full Transformer architecture,

As you can see, the Encoder consists of a Multi-Head Attention layer and a Feed Forward layer, with the input being token embeddings and positional encodings added together. I’ll try to provide an explanation of each of the components of the Encoder to the best of my ability.

The token embeddings is simply an embedding layer trained meant to take in a sequence with dimensions equal to the vocabulary size, and provide us with a lower-dimensional representation, an embedding, of the input, but thats not the entirety of it. Since we know that the positions of words in a sentence can change the meaning of that sentence, we need a way to keep track of that positional information so we don’t produce the same embedding for a word regardless of its location in the input sentence, this is the purpose of positional embeddings and positional encodings.

The diagram above uses positional encodings, which are static, for the model I used I opted for positonal embeddings, which is a trained embedding layer, both of which serve the same purpose, to add positional information into our input embeddings. The following is an implementation of the input layer,

class PositionalEmbeddings(tf.keras.layers.Layer):
    def __init__(self,vocab_size,hidden_size,max_pos_emb,dropout_rate=0.3):
        super().__init__()

        self.token_emb = tf.keras.layers.Embedding(input_dim=vocab_size,
                                                  output_dim=hidden_size)
        self.pos_emb = tf.keras.layers.Embedding(input_dim=max_pos_emb,
                                                 output_dim=hidden_size)
        self.ln = tf.keras.layers.LayerNormalization()
        self.dropout = tf.keras.layers.Dropout(dropout_rate)
    
    def call(self,x):
        seq_length = x.shape[-1] 
        pos_ids = tf.range(0,seq_length,delta=1)

        token_emb = self.token_emb(x)
        pos_emb = self.pos_emb(pos_ids)

        emb = tf.add(token_emb,pos_emb)
        emb = self.ln(emb)
        emb = self.dropout(emb)
        return emb

After that layer, we have the Multi-Head Attention and Feed Forward layers, which make up an Encoder layer. To understand the Multi-Head Attention layer, you need to have some understanding of Self-Attention. The Attention mechanism informs a model on how an input-sequence relates to an output-sequence, it is essentially a softmax layer multiplied with the output of the encoder in a sequence model, Self-Attention informs a model on how an input-sequence is related to itself, the model is learning the contextual relationships in the input-sequence.

We stack the Attention Heads because each Attention Head learns different contextual relations for the input-sequence, kind of like how in a Convolutional Neural Network each filter may represent different features of the image data, one can correspond to a wheel, another to a part of a face, etc., in a similar fashion each Attention Head can learn different features of the input-sequence. In the case of language modeling, each head might correspond to a different feature of the target language. The feed forward layer consists of dense layers that output fixed sized encodings. The following is the implementation of the Encoder layer,

class EncoderLayer(tf.keras.layers.Layer):
    def __init__(self,embed_dim, num_heads, ffn_dim, dropout_rate=0.5):
        super().__init__()

        self.mha = tf.keras.layers.MultiHeadAttention(num_heads=num_heads,
                                                      key_dim=embed_dim)
        self.ffn = tf.keras.Sequential(
           [ tf.keras.layers.Dense(units=ffn_dim,activation='relu'),
            tf.keras.layers.Dense(units=embed_dim)]
        )

        self.ln1 = tf.keras.layers.LayerNormalization()
        self.ln2 = tf.keras.layers.LayerNormalization()
        self.dropout1 = tf.keras.layers.Dropout(dropout_rate)
        self.dropout2 = tf.keras.layers.Dropout(dropout_rate)
    
    def call(self,x):
        attn_output = self.mha(x,x)
        attn_output = self.dropout1(attn_output)
        out = self.ln1(tf.add(x,attn_output))
        
        ffn_out = self.ffn(out)
        ffn_out = self.dropout2(ffn_out)

        return self.ln2(out+ffn_out)

We then put this all together to make our Transformer Encoder,

class TransformerEncoder(tf.keras.layers.Layer):
    def __init__(self,num_layers,emb_params,enc_params):
        super().__init__()

        self.emb = PositionalEmbeddings(**emb_params)
        self.enc_layers = [EncoderLayer(**enc_params)
                           for _ in range(num_layers)]
        
    def call(self,x):
        x = self.emb(x)
        for layer in self.enc_layers:
            x = layer(x)
        
        return x

Training The Model

We begin by tokenizing our data, building the datasets, putting together our model and initializing our hyperparameters, then training the model,

tokenizer = tf.keras.preprocessing.text.Tokenizer()
tokenizer.fit_on_texts(train_data['user_review'])

text_sequences = tokenizer.texts_to_sequences(train_data['user_review'])
text_sequences = tf.keras.preprocessing.sequence.pad_sequences(text_sequences,maxlen=200)

len_seq = len(text_sequences[0])
num_seq = len(text_sequences)

word2idx = tokenizer.word_index
idx2word = {v:w for w,v in word2idx.items()}
word2idx['PAD'] = 0
idx2word[0] = 'PAD'

labels = tf.keras.utils.to_categorical(train_data['user_suggestion'],num_classes=2)

BATCH_SIZE = 64
BUFFER_SIZE = 10000

dataset = tf.data.Dataset.from_tensor_slices((text_sequences,labels))
train_dataset = dataset.take(int(num_seq*0.9)).shuffle(BUFFER_SIZE).batch(BATCH_SIZE)
val_dataset = dataset.skip(int(num_seq*0.9)).shuffle(BUFFER_SIZE).batch(BATCH_SIZE)

hidden_size = 8
ffn_dim = 4
num_heads = 2
max_len = 200
num_layers = 1

emb_params = {
    'vocab_size':len(word2idx),
    'hidden_size':hidden_size,
    'max_pos_emb':max_len
}

enc_params = {
    'embed_dim':hidden_size,
    'num_heads':num_heads,
    'ffn_dim':ffn_dim
}

enc_model = tf.keras.Sequential()
enc_model.add(tf.keras.layers.Input(shape=(max_len,)))
enc_model.add(TransformerEncoder(num_layers,emb_params,enc_params))
enc_model.add(tf.keras.layers.Flatten())
enc_model.add(tf.keras.layers.Dense(units=2,activation='sigmoid'))

enc_model.compile(loss='binary_crossentropy',optimizer='adam',
                  metrics=['accuracy'])

enc_model.fit(train_dataset,validation_data=val_dataset,epochs=10)

The model achieved an accuracy of 99% on the training data, however it seemed to be stuck around 75% accuracy on the validation set, which is a clear sign of overfitting.

I had tuned the hyperparameters several times, including the dropout rates in the positional embedding and encoder layers, however the validation accuracy did not seem to improve. In the next iteration I’ll go over using the pretrained model DistilBERT for this task.
