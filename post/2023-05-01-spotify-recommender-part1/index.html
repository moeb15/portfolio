Spotify Recommender System - Part 1

Creating a Recommender System Using Spotify Data

 Posted on May 1, 2023  |  2 minutes  |  352 words  |  Mujtaba Bader

Project Overview

For this project I’m trying to build a recommender system that will recommend users a selection of artist given certain genres that the user selects. I’d like to make it a web application that users can interact with and not just some code that lies in a github repository. For this post I’ll be going over the process I followed for data sourcing and data collection.

Data Sourcing

Given the objective of this project I asked myself the following questions,

  • What exact data do I need?
  • Should I use an existing dataset from somewhere like Kaggle?
  • Should I use another method of obtaining the data like through an API?

For the 3 questions I came to the following answers, the data I needs to contain information about an artist and the music they produce, mainly just their stage name and the genres of music they belong to, and since I wanted the most recent data I decided its best not to use
a dataset from Kaggle, but instead to use the Spotify Web API to fetch the latest information on the top artists of each genre.

Data Collection

With the help of a tutorial from Tech With Tim, and the Spotify Web API documentation, I wrote the helper functions necessary to fetch the data. Here’s some of the code I used, this particular function fetches artists of a given genre with a maximum of 50 per page available and appends them into a dictionary with their name as the key, and the list of genres they belong to as the value,


def spotify_search(token,genre,i,limit=50):
    offset = limit*i
    url = 'https://api.spotify.com/v1/search'
    headers = get_auth_header(token)
    query_str = f'?q=genre:{genre}&type=artist&market=CA&limit={limit}&offset={offset}'
    query_url = url + query_str
    artist_search = requests.get(query_url,headers=headers)

    
    name_genres = {}
    try:
        for k in range(len(artist_search.json()['artists']['items'])):
            name = artist_search.json()['artists']['items'][k]['name']
            genres = artist_search.json()['artists']['items'][k]['genres']
            name_genres[name] = genres
    except KeyError:
        name_genres = {}

    return name_genres

I used another helper function to obtain the list of genres and let the code run for sometime with ratelimiting applied to avoid sending too many request in a short period of time. In the next post I’ll go over the EDA process.
