Spotify Recommender System - Part 2

Creating a Recommender System Using Spotify Data

 Posted on May 2, 2023  |  5 minutes  |  910 words  |  Mujtaba Bader

Project Overview

For this project I’m trying to build a recommender system that will recommend users a selection of artist given certain genres that the user selects. I’d like to make it a web application that users can interact with and not just some code that lies in a github repository. In the previous post I went over the data sourcing and data collection process, in this post I’ll be going over the EDA process.

Cleaning and Validating

Using the general guidelines I learned from the Google Advanced Data Analytics specializations, past projects, and online resources, I went through the process of cleaning, validating, and discovering insights from the data.

I started by creating a copy of the table stored in the database just incase I made any mistakes while transforming the data, then moved onto validating the the datatypes were correct. In the process I noticed that the genres column had datatype TEXT and not TEXT[] and transformed the column to the appropriate datatype,

ALTER TABLE ag_data_copy
ALTER COLUMN "Genres" TYPE text[] USING "Genres"::text[];

Afterwards I created another table using the artist column from the previous table and a subgenres column that consisted of the unnested text
arrays that had been stored in the genres column,

DROP TABLE IF EXISTS ag_data_flat;

CREATE TABLE ag_data_flat AS (
    SELECT "Artist", "Subgenre"
    FROM ag_data_copy, unnest("Genres") AS "Subgenre"
);

Once I had finished transforming my data I checked for any missing and duplicate values,

SELECT "Artist", "Subgenre"
FROM ag_data_flat
WHERE "Artist" IS NULL OR 
"Subgenre" IS NULL;

SELECT "Artist", "Subgenre", COUNT(*)
FROM ag_data_flat
GROUP BY "Artist", "Subgenre"
HAVING COUNT(*) > 1;

Once I had completed this step I transfered the data from a PostgreSQL database to a Pandas DataFrame for further analysis using some of Pandas functions, and for creating visualizations using Seaborn and Matplotlib.

Analysis

I started by looking at the descriptive statistics using the describe function, which provided me with a summary of the count and most occuring value in each of the columns, along with some other statistics, then moved onto checking the amount of unique values per column. Afterwards, I plotted the frequency distributions using Seaborn to better understand how the data is distributed for each genre/artist, here’s the frequency distribution for the artists and subgenres respectively,

After plotting the frequency distributions, I needed information to understand the spread of the data and to see whether there had been a significant amount of outliers within the data, I decided to create 2 new DataFrames using the group by function and then plot boxenplots coresponding to the 2, heres the code I used for this and the plots for each respectively,

genre_counts = ag_df.groupby(['Subgenre'])['Subgenre'].count()
artists_counts = ag_df.groupby(['Artist'])['Artist'].count()

Given the objective of the project and the nature of the data, it did not make sense to remove all the outliers present, the outliers in the subgenre column simply displayed what were the most popular genres and removing that would have a negative effect on the recommender system I’m trying to create, and the outliers in the artist column made no sense to remove for a similar reason, as a result I had kept all the data. Although it may be necessary to remove the subgenre of “sleep” as, I assume, its not likely that a user is going to be looking for anything in this genre and it is a signficant outlier in our dataset so it may cause issues when we train our model.

Model Selection

Basing our recommendations off of the similarities between the collected data, artists and genres, and data collected from the user, genres of interest, is known as content-based filtering, another possible approach we could take is collaborative filtering. In collaborative filtering we’d use the preferences of similar users to make recommendations for a user, so we can think about content-based filtering as an unsupervised clustering problem, and collaborative filtering as a supervised classification problem.

After going through the analysis, we see that content-based filtering is a reasobale approach for our problem, as a result we require an unsupervised learning algorithm, in particular we need to make use of unsupervised clustering algorithms, seeing as we’re trying to group together similar artists based of the genres of music they belong to, and select relevant artists based of the genres a user selects. As such, I took a look at the following clustering algorithms

  • K-Means
  • Gaussian Mixtures
  • DBSCAN

Currently I’m still in the process of deciding which model is well suited for our problem, there exists a model known as K-Modes that operates on categorical data which may be better suited for our problem than the other listed models.

If we were to take the collaborative filtering approach, we could treat the artists as class labels with our input data being the genres, in this case the user would still have to select genres of interest, however instead of clustering artists based off genre then selecting the ones closest to the genres a user selects, we’d take in the selected genres and calculate the probability that the user is selecting
a certain artist from a list of artist, then selecting the top n artist based off the probabilities, possible algorthms we could use are

  • Decision Trees
  • Random Forests
  • XGBoost

Keep in mind that neither lists are comprehensive, since this process is iterative it’s likely I’ll comeback and find a better model for the problem. In the next post I’ll go over preprocessing the data and training the model.
