Creating a Recommender System Using Spotify Data

 Posted on May 6, 2023  |  3 minutes  |  449 words  |  Mujtaba Bader

Project Overview

For this project I’m trying to build a recommender system that will recommend users a selection of artist given certain genres that the user selects. I’d like to make it a web application that users can interact with and not just some code that lies in a github repository. In the previous post I went over the EDA process and model selection, in this post I’ll finish up my discussion on model selection, and talk about preprocessing the data and training the selected model.

Model Selection

In the previous post I went over a few possible models that could be used for the recommender system, after some more research and experimentation I came to the conclusion that it was best to go with K-Modes as the model. Given that we’re dealing with categorical data, it made sense to select K-Modes over K-Means or DBSCAN. For anyone reading this who may be unfamiliar with K-Modes, here’s how it works step by step,

  • Pick k observations at random, these will act as the centers of your clusters
  • Calculate the total mismatches and assign each observation to its closest cluster
  • Define new modes for the clusters
  • Repeat the first 2 steps

If you’d like to read more on K-Modes, this post by Harika Bonthu on Analytics Vidhya should provide you with more insights.

Data Preprocessing

The orignal format of the data was the transactional format, which as we had it was Aritst: text, Genres: text[], the assumption I made was that it would be better to have the data in a tabular format by unnesting the text arrays in the Genres column, this turned out to cause some problems during training, which provided me with some undesireable results. So after experimenting, as part of the preprocessing step I put the data back into transactional format. In addition to changing the format of the data, it was necessary to encode the data before passing fitting the model, to accomplish this task I opted for using Binary Encoding over One-Hot Encoding, seeing as the data contained many unique values and given that One-Hot Encoding is not memory efficient, our data would be too large to store in memory. After encoding the data, I moved onto training the model.

def recommend_artists(genre_lst,model,encoder,data,rec_count):
    genre_df = pd.DataFrame({'Subgenre':[genre_lst]})
    genre_df['Subgenre'] = genre_df['Subgenre'].astype('string')
    genre_enc = encoder.transform(genre_df['Subgenre'])

    user_cluster = model.predict(genre_enc)
    user_recommendations = data[model.labels_ == user_cluster]['Artist'].to_numpy()
    idx = np.random.randint(0,len(user_recommendations),rec_count)
    return user_recommendations[idx]
